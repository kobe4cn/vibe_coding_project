核心能力

可组合的标签与票据操作

票据 CRUD、标签体系、状态机、搜索筛选与附件上传，全部一键可用。

票据生命周期全覆盖

  • 创建、编辑、删除、完成、取消完成一气呵成。
  • 支持分页排序，列表秒开，适配移动端。
  • 状态流转记录可审计，团队协作更透明。

16 预设标签 + 自定义扩展

  • 状态、优先级、模块、工作流多维度组合。
  • 标签可设置颜色与图标，兼容自定义。
  • 标签-票据关联一键增删，聚类视图清晰。

安全的有限状态机

  • open → in_progress → completed/cancelled 路径受控。
  • 完成需要填写处理结果，避免信息缺失。
  • 任意状态可 reopen 到 open，方便回溯。

搜索、筛选与附件

  • 按标题搜索，状态/优先级/标签多维筛选。
  • 附件多格式上传，单文件 10MB 内自动校验。
  • 多字段排序 + 分页，列表操作流畅。

技术栈

后端高性能，前端高体验

Rust + Axum + PostgreSQL 的稳定性，React + Vite + Tailwind 的敏捷性，共同支撑票据全链路。

后端

Rust · Axum · SQLx

  • Tokio 异步，Tracing 链路日志。
  • SQLx + PostgreSQL，类型安全的查询。
  • 健康检查/就绪检查开箱即用。

前端

React 19 · Vite 7 · Tailwind 4

  • TanStack Query 管理异步数据。
  • React Router 7 路由与数据协同。
  • Shadcn 组件 & Zod 表单校验。

部署

Docker Compose 一键启动

  • 内置 docker-compose，前后端 + 数据库一键拉起。
  • 环境变量模板随附，复制即用。
  • 产出 dist & 镜像，适配反向代理。
开始部署

快速开始

两种方式，10 分钟跑起来

方式一：Docker 推荐

docker compose up -d open http://localhost

方式二：本地开发

cd backend && cp env.template .env && cargo run cd frontend && npm install && npm run dev
